# 公益站签到机器人 — 项目计划书

## 一、项目目标

构建一个部署在 VPS 上的 Telegram Bot，自动签到 15+ 个 NewAPI 公益站点，通过按钮交互提供状态查询、API Key 管理、模型查看等功能。

## 三、核心功能模块

### 模块 A：按钮式交互界面（TG Bot）

#### A1. 状态总览 `/s`
- 用 InlineKeyboardButton 渲染全部站点
- 每行 3 个按钮，每个按钮格式：
  ```
  [✅图标] [站点图标] [站点简称]
  ```
- 按钮颜色 / 前缀取决于状态：
  - `✅` = 今日已签
  - `⚠️` = 未签到
  - `❌` = 未认证/错误
- 底部统一控制栏：全部签到按钮 + 刷新按钮

#### A2. 站点详情（点击按钮）
- 弹出新消息，显示完整信息：

```
┌─────────────────────────────────┐
│ [状态] [图标] 站点名             │
│ 🔗 URL                          │
│                                 │
│ 💰 余额: $xx.xx                 │
│ 👤 用户: YuUu (ID: 1555)        │
│ 📅 签到: ✅ 今日已签 | 累计 X 天 │
│                                 │
│ 🔑 API Key:                     │
│ sk-xxxx... (code格式/可复制)     │
│                                 │
│ 🤖 模型 (12个):                 │
│ deepseek-v4-pro, glm-5.1, ...   │
└─────────────────────────────────┘
```

#### A3. 命令列表

| 命令 | 别称 | 功能 |
|------|------|------|
| `/start` | — | 欢迎页 + 命令提示 |
| `/s` | `/status` | 按钮式状态概览 |
| `/d` | `/detail` | 显示某站点详情（无参数则按钮选择） |
| `/c` | `/checkin` | 一键全部签到 |
| `/man` | `/manual` | 选择单个站点手动签到 |
| `/b` | `/balance` | 余额汇总 |
| `/k` | `/keys` | 全部 API Key 列表（可复制） |
| `/t` | `/time` | 查询下次自动签到时间 |
| `/log` | — | 最近日志 |
| `/settime` | — | 手动设定签到时间 (HH:MM) |

---

### 模块 B：自动签到引擎

#### B1. 定时策略
- 每日 **8:00~20:00 随机时间**（秒级精度）
- 启动时生成当日随机时间，保存到 `.next_checkin` 文件
- Bot 每 5 分钟检查一次是否到达预定时间
- 签到完成后自动生成第二天的随机时间
- 手动 `/c` 签到不影响自动定时
- `/settime HH:MM` 可手动设定签到时间

#### B2. 签到流程（每站点）
```
1. 检查是否有 session cookie → 验证 /api/user/self
2. 如果失效但有 login 配置 → 自动登录
3. 调用 /api/user/checkin?month=YYYY-MM → 检查今日是否已签到
4. 若未签到 → POST /api/user/checkin → 执行签到
5. 记录结果到日志
```

#### B3. 站点类型支持
- `new_api` — 标准 NewAPI 站点（15个）
- `login_reward` — 访问即奖励（AnyRouter）
- `sub2api` — Sub2API 协议（需 Authorization header）

---

### 模块 C：认证管理

#### C1. 自动登录（支持 10 个站点）
- 用户名/密码 → POST `/api/user/login`
- 获取 session cookie + user_id
- 自动续期（cookie 过期后重登）

#### C2. 手动 Cookie（4 个站点）
- 从 Chrome DevTools 粘贴 session cookie
- Cookie 过期后 TG Bot 提示"需要更新 Cookie"

#### C3. 不支持自动登录（3 个待处理）
- AIHub: 需要 Turnstile 验证码
- ChuYel: 密码登录被管理员禁用
- Sub2API: 需要 Authorization token
